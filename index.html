Entwicklungsumgebung

In dieser Dokumentation werde ich viel über HTML und CSS Dokumentieren, damit wir aber überhaupt beginnen können unsere eigene Website zu erstellen, brauchen wir eine Entwicklungsumgebung mit gewissen Tools. Und diese Tools wären:

  • Ein HTML/CSS Editor.
    • unserem Fall Visual Studio Code.
  • Ein Bildbearbeitungsprogramm.
    • z.B. Photoshop oder Gimp.
  • Einen Browser
    • z.B. Firefox, Chrom, Opera.
  • FTP Client
    • z.B. FileZilla, Cyberduck.
  • Nachschlagewerk
    • z.B. W3schools.
  • Webserver
    • In unserem Fall XAMPP.
Wichtig ist zu wissen, dass wenn man eine Webseite schreibt, dass man auch die alle Aktuellen Browser sich Installiert und darauf seine Website testet.

HTML

Element

Html besteht aus einem Html, head, und body Tag. so wird eine Website Gegliedert was den Text betrifft. Wichtig ist es auch noch zu wissen aus was eine HTML Zeile besteht. nämlich aus einem sich öffnenden und aus einem sich schliessenden Tag. z.B <p> </p> jedoch fehlen noch ein paar Sachen, denn man kann einem Tag auch einen Inhalt geben z.B <p> Das ist der Inhalt welcher nachher auf der Website angezeigt wird. </p>. Nebst dem Inhalt kann man auch noch einen Attribut wert eingeben. Der Attributname sagt aus, welche Art von Zusatzinformationen über den Inhalt des Elements Sie damit angeben. Er sollte in Kleinbuchstaben geschrieben werden. z.B <p lang="de-ch"> steht für die Sprache Deutsch Schweiz</p>

Text

Einen normalen Text in HTML schreibt man mit dem <p> Tag. Fast alle Texte auf dieser Seite wurden mit einem p Tag geschrieben. Das p bedeutet soviel wie Paragraph. Zu den Texten gehören auch die Überschriften, welche mit dem <h1> bis <h6> Tag nummeriert werden. Wichtig zu beachten ist, dass alle Text-Tags auch einen End-Tag brauchen. Zudem kann man noch andere Texte z.B Fett kursiv oder auch in anderen Formen schreibern jedoch empfiehlt es sich dies im CSS zu tun.

Besondere Markups

Mit dem Titel Markup betrachtet man verschiedene Informationen über das gesamte Dokument oder nur über ein spezielles Element. Beim Dokument wird mit <!DOCTYPE html> gezeigt, dass die Seite / das Dokument mit HTML 5 geschrieben ist. Kommentare kann man auch in HTLM schreiben, in dem man <!--Hier steht der Kommentar--> schreibt. In den meisten Editoren wird das dann in einer speziellen Farbe angezeigt. Für die genauere ansprechung der Elemente kann man mit KLassen oder IDs arbeiten. Eine Klasse (class) wird für mehrere Objekte verwendet, wobei ID nur für ein Elemnt gilt. Klassen und IDs werden als Attribut gschrieben. Das Arbeiten mit Klasse und IDs macht aber erst dann Sinn, wenn man auch CSS ins Spiel bringt. Zum Markup gehören auch DIVs, welche zum gruppieren von Inhlaten dienen. Auf dieser Seite sind immer eine Überschrift und ein Text im selben DIV. Auch die meta-Tags gehören in dieses Kapitel. Sie dienen dazu, Informationen über die ganze Seite zu speichern, zum beispiel das Charset, welches den Zeichensatz angibt, in welchem die Website geschreiben ist. Meta-Tags sind auch leere Elemente.

Listen

Es gibt mehrere Arten von Listen: die geordnete Liste <ol>, die ungeordnete Liste <ul> und dann gibt es noch die Definitionsliste <dl>. In den Listen-Elementen kommen dann noch die <li> Tags ausser bei den Definitionslisten, bei ihnen kommt ein <dd> Tag hinein. Man kann auch Listenüberschriften machen (nur bei Definitionslisten), mit dem Tag <dt>. Man kann auch Listen ineinander verschachteln, damit die Punkte sowie Unterpunkte haben. Hier sind die Dropdownmenüs als Liste erstellt worden.

Formulare

Fomrulare werden für Eingabefelder, Checkboxen und Mehrfachauswahlslisten. Man umgibt alle Fomrulare mit dem <form>-Tag. Ein Eingabefeld macht man mit dem Input-Tag, dort kann man auch noch die maxlenght (maximale Länge) angeben. Mit dem type="password" kann man die Eingabe mit Punkten überdecken, wie man das halt schon kennt aus anderen Webseiten, wo es ein Login-Fenster hat. Hier ein Beispiel: Mit einer Textarea kann man einen ganzen Bereich einfügen, in welchen man dann schreiben kann. Mit Javascript könnte man dann diese Inhalte in eine Datenbank eintragen lassen. Man kann auch mit dem input type="radio" eine auswahl erstellen, bei denen man nur ein Punkt auswählen kann. Mit dem Typ checkbox kann man eine Liste erstellen, wo man mehrere Items auswählen kann. Mit <select> kann man eine Selektionsliste erstellen: Mit input type="submit" kann man einen Button erstellen, welcher nur dazu dient, die Inhalte einzusenden. Man kann aber auch einen Knopf einfügen, dies kann man erreichen durch das Element Mit <label> kann man eine art Div für mehrere Felder erstellen. Mit <fieldset> kann man Formulare gruppieren, so hat man auch einen schönen Rahmen. Mit dem Typ date kann man eine Datumseingabe erstellen. In HTML ist auch schon eine Funktion für Suchefelder vorhanden, mit diesen Feldern kann man noch nichts Suchen, aber man hat schon das gnaze Layout des Feldes. Für diese Felder wählt man als Typ den Wert search aus. Mit einem Placeholder kann man einen grauhinterlegeten Text einfügen, genau wie beim Suchfeld.

Flash Video und Audio

Mit Flash kann man Animationen erstellen, dies wurde anfangs der 90er Jahre gemacht, heute macht man es nicht mehr so häufig. Eine Flash-Animation fügt man mit Javascrip ein. Ein Video fügt man mit dem <video> Element ein. Für Videos gibt es noch attribute wie preload, welcher das video schon lädt, bevor man es wiedergibt. Mit auto lädt der Browser das Video mit der Seite herunter, mit autoplay kann man das Video automatisch abspielen lassen und mit loop wird das Video immer wiederholt.

Mit dem <audio> kann man ein Audiofile in die Webseite einbauen, man kann dabei auch die gleichen Attribute verwenden, wie schon bei den Videos.

Bilder

Bilder kann man in HTML mit dem <img src="link" alt="Bild"/> einfügen. Src steht für Source, das ist der Speicherort des Bildes (kann auch ein Link zu einem Bild sein, welches im internet ist). Das Alt-Attribut ist auc hsehr wichtig, denn Bildschirmleser lesen diese Alt-Texte, welche beschreiben, was im Bild zu sehen ist. Auch Suchmaschienen wenden diese Technik an und können so den Inhalt der Seite bestimmen. Die grosse Arbeit an den Bilder kommt dann mit CSS ins Spiel.

Links

Links braucht man, wenn man zum Beispiel andere Seiten oder Orte auf der selben Seite verlinken will. Sie werden mit dem <a> Tag geschrieben. Als Attribut wird href="link" verwendet, zwischen die a-Tags kommt der Text, auf den man klicken kann. Als weiteren Attribut kann man noch target="_blank" schreiben, dieser Wert bestimmt, dass die Seite in einem neuen Tab geöffnet wird. Auf dieser Seite habe ich Links verwendet, um von der Navigation aus zu den verschiedenen Punkten zu springen.Google

Es gibt zwei Arten von Pfaden: den Absoluten und den Relativen Pfad. Der Absolute ist genau so, wie er angegeben ist, wird zum Beispiel für externe Links verwendet, wie Internetseiten. Relative Pfade werden für Speicherorte, welche intern sind, verwendet, falls man die Internetseite verschiebt, hat man keinen grossen Aufwand, alle Pfade zu ändern. Ein relativer Pfad ist immer von dem speicherort der offenen Datei abhängig, also kann man mit /unterorden/zweite_seite.html in einen Unterordner gelangen, will man aber in einen Elternordner gehen, so gibt man vor den Pfad einfach noch ../ an. Damit kommt man einen Ordner weiter raus, dann kann man wieder in einen anderen Ordner navigieren.

Tabellen

Eine Tabelle erstellt man mit dem <table>-Tag. Eine neue Zeile definiert man mit einem <tr>-Tag. Tr bedeutet Table-Row (Tabellenreihe). Eine Zelle schreibt man mit dem <td>-Tag. Wenn man td mit th ersetzt, dann hat man eine Tabellenüberschrift. Wenn man eine Zelle über mehrere Zellen strecken will, dann wählt man für die horizontale Ausrichtung colspan und für die vertikale Ausrichtung dan rowspan Attribut. Man kann eine Tabelle auch in mehrere Teile unterteilen, genau wie ein HTML dokument. Mit <thead> legt man einen Tabellenkopf an, mit <tbody> legt man den Tabellenbody fest und mit <tfoot> die Fusszeile /n. Das ist zum Beispiel wichtig, wenn eine Tabelle viele Zeilen besitzt, dass beim Scrollen die Kopf- und Fusszeile auf dem Bildschirm fixiert werden, das sorgt für eine gute Übersicht. Mit den Attributen cellpadding und cellspacing kann man den Innenraum der Zellen und den Absctand zwischen ihnen bestimmen. Man kann auch den Rand einzelner Zellen färben, sowie den hintergrund. CSS

CSS wird gebraucht damit man seine geschrieben HTML Zeilen designen kann. durch CSS erhält eine Website unter anderem auch eine Strucktur.

CSS einbinden

CSS kann verschieden in HTML integriert werden. Man kann z.B. Inline CSS anwenden, bei dem man einem Tag den Attribut Style hinzufügt. jodoch gibt es eine weitere Möglichkiet im Head mit dem <Style>-Tag zu arbeiten, indem man dann seinen CSS-Code hineinschreibt. Die beste Möglichkeit jedoch CSS fürs HTML zu bekommen besteht durch das einbinden einer Datei. die Datei benennt man wie folg:style.css . In dieser Datei steht dann der CSS-Code. Diese bindet man im Head-Bereich mit dem <link href="style.css" rel="stylesheet" type="text/css"/> Tag ein. Als Attribute fügt man wie hier noch den Link oder den Dateinnamen ein, mit rel bestimmt man, um was es sich handelt und mit type bestimmt man, in welcher Form der Text geschrieben ist.

CSS Regeln

Genau wie bei HTML haben CSS Regeln auch eine Strucktur. SIe bestehen aus einem Selektor und einer Deklaration ,wobei der Selektor in der Regel angiebt auf welches Element die Regel angewant werden sollte. Und eine Dekleration besagt, wie die im Selektor genannten Listen-Elementen Formnatiert werden sollen. Die Deklarationen selbst bestehen wiederum aus 2 Teilen nämlich aus Eigenschaft und Wert, wobei die Eigenschaft angiebt was genau gemacht werden sollte und der Wert z.B die grösse einer schrift mit z.B 20px besagt.

p {

   font-size: 12px;

}

Selektor

Es gibt mehrere Arten von Selektoren:

Universalselektor
Gilt für alle Elemente auf der Seite.
Typselektor
Gilt für ein bestimmes Element h1 h2 p div usw.
Klassebselektor
Dieser Selektor gilt für alle Elemente mit dem selben Wert für den Attribut class. Also zum Beispiel für alle Elemente der Klasse alledivs, diese Klassen werden mit einem Punkt im CSS-Code angesprochen, das sieht so aus: .alledivs.
ID-Selektor
Er bewirkt das gleiche wie der Klassenselektor, nur halt für ID's. Im CSS sieht das dann so aus: #div1. Hier wird die ID div1 angesprochen.
Kindselektor
Dieser Selektor gilt für alle Elemente, welche dem Angegebenen Elemente unmitelbar untergeordnet sind. Wenn man zum Beispiel alle Links in einem Listenelement ansprechen will, sieht es so aus: li>a.
Nachkommensselektor
Er funktioniert gleich wie der Kindselektor, ausser dass hier noch andere Elemente vor dem Ausgewälten kommen können. Für genau das gleiche beispiel sieht das dann so aus: li a.
Selektor für benachbarte Geschwister
Dieser Selektor gilt für ein Element, welches dem Angegebenen direkt gleichgeordnet ist. Also für für das nächste Div nach einem Div sieht das so aus: #div1+#div2.
Allgemeine Geschwisterselektor
Gilt für alle Elemente, welche dem Angegebenen Element gleichgeordnet sind. Für alle Div's im Body würde man das also so schreiben: body~div.

Text

es gibt verschiedene Möglichkeiten einen Text in CSS zu berarbeiten, man kann z.B. die Farbe ändern, die grösse ,oder auch wie man einen Text dekorieren will. jedoch ist es am besten für Webseiten, wenn man sich für eine Serifenlose Schrift entscheidet. Um bei einem text die Schriftart zu verändern benutzt man die Eigenschaft font-family. Meistens verwendet man, um eine Schriftart zu ändern, Google Fonts. die Schriftgrösse ändert scih furch die Eigenschaft font-size, welche in Pixel oder Prozent angegeben werden kann. Damit die schrift etwas Farbe bekommt benutzt man die eigenschaft color. Um einen Text leicht Fett anzuzeigen benutzt man die EIgenschaft font-weight: bold; und um einen text kursiv anzuzeigen benutzt man font-style: italic;. es gibt noch weitere Möglichkeiten einen Text zu Dekorieren und zu verschönern, nämlich durch die Eigenschaft text-decoration, mit dieser Eigenschaft kann man einen text unter, über oder durchstreichen. eine Linien höhe wird durch die Eigenschaft line-height verändert. Um einen grösseren Abstand zwischen den Zeichen zu erlangen, verwendet man word-spacing. Wichtig bei einem Text ist natürlich noch die Ausrichtung, welche mit text-align: center; z.B beinflusst werden kann.

Google Fonts

google fonts können Schriftarten im Internet ausgesucht werden und mit CSS eingebunden werden. Man sollte jedoch darauf achten, dass die Gewählte Schrift Mit jedem Gerät kompatiebel ist. wenn man eine Schrift nun ausgewählt hat und sich für den Download entscheidet, kann man die Schrift durch @font-face {} einbinden. Es gibt jedoch auch eine Andere Möglichkeit eine Schriftart von Google fonts einzubinden nämlich durch einen Link im HTML code welcher z.B Lautet: link href="https://fonts.googleapis.com/css?family=Acme" rel="stylesheet". um bei Goolge Fonts eine Schriftart zu benutzen braucht man keine Lizenz, es ist also gratis.

Farbe

Farben spielen bei Texten und Buchstaben bei Websteiten eine grosse Rolle. Um eine Farbe zu verändern, verwendet man color als Eigenschaft. man kann den Wert in RGB, RGBA oder durch eine Hexadezimale angeben. bei Farben welche eher bekannter sind, wie z.B rot reicht es auch einfach die Farbe einzugeben. Um eine Hintergrundfarbe zu ändern benutzt man die Eigenschaft Background-Color. wenn man will das eine Schrift Transparent ist, kann dies mit rgba oder der Eigenschaft opacity machen.

Kästen

Bei CSS sind mit Kästen die div Elemente in einem Html code gemeint. Standartmässig ist ein Kasten (div) gerade gross genug für seinen Inhalt. Mit den EIgenschfaten height und width kann man Abmessungen selbst festlegen. Manche Seiten dind so gestaltet, dass sich ihre gtösse an die des Benutzerbildschirmes anpassen. Dabei wird die Eigenschaft min-width und max-width verwendet. die hähe wird demnach auch mit min-height und max-height festgelegt. Man kann mit CSS auch überlaufende Inhalte anzeigen, denn die Eigenschaft overflow sagt dem Browser, was er tun soll, wenn der Inhalt eines Kastens grösser ist als der Kasten selbst. man kann folgende zwei Werte benutzen, nämlich scroll und hidden. mit hidden wird der zusätzliche Inhalt welcher zu gross für den Kasten ist einfach verborgen und mit scroll wird der Kasten mit einem Rollbalken versehen, welcher es ermöglicht durch den Kasten zu scrollen.

Listen

Mit der Eigenschaft list-style-type bestimmt man die Form oder Gestalt von aufzählungspunkten. als wert kann man dabei z.B. disc, circle, cquare oder auch upper-roman wählen um die Aufzählungszeichen zu ändern. um ein Bild als Aufzählungszeichen zu Nehmen benutz man die Eigenschaft list-style-image und als Wert gibt man eine URL ein. Listen werden standartmässig eingerückt, wobei die Eigenschaft list-style-position angiebt, ob der AUfzählungspunkt innerhalb oder ausserhalb des kastens mit den einträgen stehen sollen. Die Eigenschaft kennt demnach zwei Werte, nämlcih outside und inside. die Begriffe sind selbsterklärend. Zudem gibt es noch eine Kurzschreibweise für Listen in CSS. welche mit list-style deklariert werden.

Tabellen

Mit der Eigenschaft width legt man die Breite einer Tabelle fest. Mit der Eigenschaft padding legt man den Abstand zwischen dem Rahmen und den einzelnen Tabellenzellen und ihrem Inhalt fest. mit der Eigenschaft text-transform wandelt man den Inhalt der Tabellenkopfzeilen in Grossbuchstaben um.letter-spacing und font-size sorgen zusätzlich für die Formatierung des Inhaltes von Tabellenkopfzeilen. text-align Richtet den Text aus und backround color ändert die Farbe. Mit : hover hebt men eine Tabelllenzeile hervor, wenn man mit dem Mauszeiger drüberfährt. Falls sich die Möglichkeit mal ergeben würde, dass sich eine leere Zelle in der Tabelle befindet kann man mit der Eigenschaft empty-cells festlegen ob die Rahmen dieser Zelle angezeigt werden sollen oder nicht. Folgende Werte können gebraucht werden show, hide und inherit. ich glaube show und hide sind klar deswegen werde ich nur inherit erklären. Also Inherit sogt dafür, dass wenn sich zwei Tabellen verschachtelt haben, der äusseren zu gehorchen. Mit der Eigenschaft border-spacing regelt man Abstände benachbarter Zellen. mit dem der Eigenschaft border spacing kann man nun diese Abstände vergrössern oder verkleinern. Der Wert wird für gewöhnlich in Pixeln angegeben. Wichtig ist zu wissen, dass wenn zwei Rahmen sich treffen das der Rahmen dann doppelt so dick wird. Dies kann man mit der Eigenschaft border-collapse zusammenfallen lassen. wenn man die Rahmen voneinander trennen will benutz man das Gegenteil nämlich border-seperate.

Box-Modell

Mit dem Box-Modell kann man sehen, wie sich die Kombination aus Rahmen, Innen- und Aussenabstand zusammensetzt. Jede Eigenschaft dieses Modelles hat 1, 2 oder 4 Werte in px angegeben. Mit einem Wert hat es auf jeder Seite den gleichen Abstand, mit 2 Werten spricht man die Ober- und Unterseite an und die Seiten links und rechts. Hat man 4 Werte so kann man auf jeder Seite einen anderen Abstand wählen, man beginnt immer mit dem Wert der Oberseite und arbeitet dann im Uhrzeigersinn. Das Box-Modell sieht so aus:

Flexbox

Flexbox bietet eine größere gestalterische Freiheit im Vergleich zum klassischen CSS-Box-Modell. Das Flexible Box Layout Module – kurz Flexbox – soll es erleichtern, Elemente vertikal oder horizontal neben- oder untereinander zu positionieren. Dabei reagieren die mit Flexbox ausgestatteten Elemente dynamisch auf Änderungen des Viewport, behalten dabei jedoch immer ihre Relation zum jeweiligen Elternelement stabil bei. Diese Aufgabe, die insbesondere mit dem Fortschreiten responsiven Designs an Bedeutung zugenommen hat, wurde bislang in aller Regel unter Verwendung der CSS-Eigenschaft float erledigt.

Die Flexbox Bearbeitung kann man mit diplay: flex aktivieren. Mit flex-direction kann man die Ausrichtung der Elemente beeinflussen, zum Beispiel bei row wird es eine Reihe, bei column eine Spalte und wenn man das -reverse daran hängt, dann wird die ganze Reihenfolge umgedreht. Mit Order kann man die Reihenfolge festlegen, man gibt als Wert einfach die gewünschte Stelle an. Mit align-item oder align-content oder align-self kann man die Anordnung und die Ausdehnung eines Elementes oder des ganzen Divs definieren. Mit justify-content kontrolliert man den Abstand zwischen den einzelnen Elementen und auch ob sie in der Mitte, rechts oder links liegen. Mit flex-wrap legt man den Umbruch der einzelnen Flex-Elemente fest.

Media Queries

Media Queries sind ganz wichtig, wenn man will, dass die Webseite auch auf Smartphones gut aussieht. Dazu nutzt man Media Queries. Man schreibt sie immer unten im CSS-File auf. Sie werden so geschrieben @media (max-width: 768px) {}. Dieses Beispiel besagt, dass wenn das Bild kleiner als 768 Pixel ist, dass dann der Inhalt der geschweiften Klammer gilt, dort schreibt man zum Beispiel hinein, dass die Schrift kleiner wird. Bei dieser Seite wird die Schrift immer kleiner und die Boxen werden immer grösser. Normal sollte man bei 768 Pixeln etwas ändern, das ist die Definierte breite für Smartphones.

Layout in HTML

Man sollte darauf achten, dass die Webseite auch strukturiert in HTML ist, also man unterteilt die Webseite in Teilabschnitte. Man Teilt auch nur den Body auf, sonst nichts. Man erstellt einen <header>, also ein Kopfbereich, in welchen dann das <nav> kommt, das ist der Tag, in welchen man dann die Navigationsleiste schreibt. Jetzt kommt der <content>, in welchen der ganze Inhalt der Webseite kommt. Diesen Bereich könnte man dann noch mit <article>-Elementen weiter aufteilen. Ganz unten auf der Webseite ist dann noch der <footer>, in welchem die Copyright-Informationen der Seite stehen. Auf gewissen Webseiten stehen noch mehr Informationen im Footer, zum Beispiel ein Link zu einem Kontaktformular und Standorte der Firma.

Testprotokoll

Wenn man mit der Webseite fertig ist, dann sollte man diese noch testen. Dafür erstellt man am besten ein Testprotokoll, auf dem oben steht, von wem getestet wurde und wann getestet wurde. Das Testprotokoll an sich teilt man in vier Spalten auf, eine indem steht was getestet wird, in der zweiten wie getestet wird, in der dritten was man erhofft (das erhoffte Ergebnis) und in der vierten Spalte das erhaltene Ergebnis und auch der Fehler, wenn einer vorhanden ist. Man könnte noch ein extra Protokoll erstellen, wo man protokolliert, wie man den Fehler behebt. Zum Testen würde ich auf jeden Fall empfehlen:

  • Browserkompatibilität (Wird die Webseite in allen Browsern korrekt angezeigt)
  • Responsives Design korrekt (Ist die Webseite responsiv, funktioniert das auch alles und wie sieht es wirklich auf einem Smartphone aus)
  • Funktionierende Links (funktionieren alle Links auf der Seite, gilt für Verlinkungen intern sowie Verlinkungen zu anderen Webseiten)
  • Wenn man JavaScript benutzt, dann muss es natürlich fehlerfrei funktionieren

Ein letzter wirklich wichtiger Punkt ist, dass der gesamte Code valide ist, mehr dazu unter Validierung.

Validierung

Es ist ganz wichtig, dass der gesamte Code der Webseite valide ist, das heisst er hat keine Fehler. Das kann man testen, indem man den gesamten Code in einen Validator reinkopiert und dann auf überprüfen drücken. Wenn der Code noch Fehler hat, dass muss man diese beheben und den Code nochmals einfügen, das solange bis er keinen Fehler mehr hat. Das macht man auch mit dem CSS-Code und falls man auch ein JavaScript hat, auch mit diesem. Hier ist noch der Validator von w3schools, mit dem ich auch immer arbeite.
